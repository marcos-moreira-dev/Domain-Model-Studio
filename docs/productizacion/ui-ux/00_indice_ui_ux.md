# Índice — UX/UI, scaffolding y herramientas del editor

Estado: **diagnóstico y criterios de alineación**  
Propósito: consolidar reglas de interfaz gráfica para que Domain Model Studio se comporte como una herramienta de diagramación sobria, operativa y verificable.

## Documentos

| Documento | Propósito |
|---|---|
| `01_criterios_toolbar_paneles_canvas.md` | Define qué va en toolbar, qué va en panel izquierdo, qué va en panel derecho y qué pertenece al canvas. |
| `02_auditoria_inconsistencias_ui_por_vista.md` | Revisa inconsistencias detectadas en capturas y código por tipo de vista/editor. |
| `03_estilo_visual_tokens_paleta_sobria.md` | Fija criterio de paleta, tokens CSS, colores sólidos, bordes y deuda del fondo gris/café. |
| `04_plan_alineacion_ui_canvas_exportacion.md` | Propone orden de implementación para alinear UX/UI, canvas común, tabs escrolleables y exportación SVG/PNG. |

## Regla rectora

```text
La interfaz no debe presumir complejidad: debe hacer evidente qué se puede crear, seleccionar, mover, editar, validar y exportar.
```
