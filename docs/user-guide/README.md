# Guía de usuario — Domain Model Studio

Esta guía explica cómo usar **Domain Model Studio** para pasar de Markdown estructurado a una salida visual o documental editable y exportable.

La guía acompaña a la guía académica integrada de la aplicación y a la ayuda operativa del SideDock. La guía académica explica teoría; el SideDock explica cómo usar la herramienta activa; esta guía revisa el flujo completo con más calma.

---

## Flujo recomendado

```txt
1. Elegir una plantilla o ejemplo oficial del tipo de diagrama.
2. Adaptar el Markdown con la información del cliente.
3. Importar el Markdown en la aplicación.
4. Revisar la salida central: diagrama, matriz, mapa, wireframe o documento.
5. Ajustar manualmente lo mínimo necesario.
6. Guardar como .dms.
7. Exportar PNG, SVG, PDF o Markdown según el tipo.
8. Reabrir el .dms cuando necesites continuar.
9. Consultar Ayuda > Guía académica cuando necesites teoría del artefacto, o el botón Ayuda del SideDock cuando necesites operación de la herramienta activa.
```

---

## Salidas principales

```txt
Modelo conceptual        → diagrama Chen / pata de gallo
Diccionario de datos     → documento PDF/Markdown
Mapa de módulos          → mapa visual
Roles y permisos         → matriz visual
Flujo de pantallas       → tarjetas conectadas
Wireframes administrativos → bocetos visuales
UML / C4 / BPMN          → diagramas visuales mínimos exportables
```

## Regla de lenguaje

La guía usa lenguaje del producto:

```txt
proyecto, modelo, diagrama, documento, matriz, wireframe, ejemplo, plantilla,
vista, propiedades, conexión, exportación y ayuda.
```

No se debe explicar el uso normal de la aplicación con jerga de desarrollo.
